# CLAUDE.md

このリポジトリで作業する際の運用ルール。設計思想やコードの詳細は `.ClaudeCode/DESIGN.md`・
`.ClaudeCode/DEV_NOTES.md`・`.ClaudeCode/PROGRESS.md` を参照。

## 開発体制・レビュー方針

- 個人開発の小規模リポジトリ。GitHub Actions上のCI（ビルド確認・テスト等）は設定していない
  （`.github/workflows/release.yml` はタグpush時のリリース物作成専用で、PRのゲートではない）。
- 第三者レビューは行わない体制のため、**PRを作成したら自分でマージしてよい**
  （draftのまま止めず、レビュー待ちにする必要はない）。単独の小規模変更であれば、
  状況に応じて直接コミットしても構わない。
- ユーザー（リポジトリ管理者）はビルド環境を持たない前提で動く。したがって
  「実装 → コミット/マージ → リリース（タグpush） → ユーザーがDLして実機検証 → フィードバックを踏まえて再編集」
  というサイクルで開発が回る。ユーザーへの確認が必要な事項（設計判断の分岐など）は、
  作業を止めて都度聞くのではなく、**リリース後にまとめて聞く**。

## セッションをまたぐ進捗管理

複数セッションにまたがる実装作業（機能追加・改善提案の実装など）は、
`.ClaudeCode/PROGRESS.md` に実装予定の内容と進捗状況を記録しながら進めること。
セッション開始時はまずこのファイルを確認し、続きから作業する。
